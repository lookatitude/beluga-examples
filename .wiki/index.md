# Beluga Examples Wiki

**Scope: examples-local.** This wiki holds knowledge specific to the runnable examples: per-example conventions, framework version compatibility notes, testing patterns, and known gotchas. It does **not** hold framework patterns (those are in `../framework/.wiki/`) or cross-repo coordination material (that is in the workspace wiki at `../.wiki/`).

## What lives here (and when)

| Topic | Status |
|---|---|
| Per-example compatibility notes (which framework version each example targets) | *on-demand* |
| Testing patterns (how CI validates examples) | *pending — when CI is added* |
| Known gotchas (API keys, rate limits, provider-specific quirks in examples) | *on-demand* |
| Example-to-tutorial linking conventions | *pending — Phase 5+* |

## Where everything else lives

- **Framework Go patterns, APIs, architecture** → `../framework/.wiki/`
- **Release flow, dispatch chains, cross-repo incidents** → workspace `../.wiki/`
- **Website content conventions, design tokens** → `../website/.wiki/`
